Install

Install Lion Admin Theme

Uploading Theme

Extract the zipped package downloaded from wrapbootstrap to your desktop, in the extracted package you will find the "Theme" folder.

  • via FTP : upload the extracted folder to /yourdomain.com/folder/ on your server.

That's it, you have installed and activated the theme.

CSS

Structure

Lion Admin Theme is a responsive layout design. It uses Twitter Bootstrap as a main framework. Please, refer to the Bootstrap page for more information about it.

Bootstrap 2.1.1 was used in this design.

Like the HTML pages, CSS files are commented too. Bootstrap comes with 2 CSS files, the main structure is located in bootstrap.css and the responsive configuration is in bootstrap-responsive.css.

In addition to this files, Lion uses a main CSS file for overall aspect. This file is "main.css". There are some other CSS files included in this theme. Below is the detailed list.

CSS List

- bootstrap.css (main structure file)
- bootstrap-responsive.css (responsive structure file)
- main.css (main design aspect)
- login.css (used only in the index.html - login page)
- error.css (used only in the 404.html error page)
- table.css (DataTable plugin - CSS configuration)
- prettyPhoto.css (prettyPhoto plugin - CSS configuration)
- fullcalendar.css (FullCalendar plugin - Used in calendar.html only)
- elfinder.css (elFinder plugin - File Manager used only in manager.html)
- open_sans.css (Open Sans Font - Google font)

Dashboard

Dashboard

dashboard.html is the main page of the theme. index.html is the login page. Please pay attention if you rename the files.

Lion is focused in a simpler design. Most admin panels have tons of features and interfaces. We try to keep it simple but elegant. The dashboard page is divided in two main parts (top and middlewrap section). This sections are subdivided in 4 blocks the top part, and 6 blocks the middlewrap section.

The first block is the user block information. It shows the image of the user, the name and the new emails and messages. This is highly customizable, so you can add whatever you want. The color and the style used in this block is defined in the dash-unit section in the main.css file.

The next 3 blocks are charts. As you can see, the charts are created using Highcharts script. Further in this file we will talk about charts and their configuration. Charts are an essencial part of an Admin Panel theme. For me, is one of the most important part, so I try to create nice looking charts. If you realise, when you configure your dashboard, stadistics and information are key parts, so charts have an important position in the layout.

The six bottom blocks shows information that you could be interested. The idea is to keep a clean design but with functionality.

Notifications

 

A notification shows when you load the page, to modify this, just edit the dash-noty.js file. Remember to add all the noty javascripts needed. In the User Interface Elements page, you have more demos and can see how to configure them.

The notifications a great tool to show the latests messages, emails, tasks and issues. To change the default color of the notes, refer to assets/js/noty/theme/default.js file. Also, you can learn more about this script in the noty homepage.

Other Scripts

Calendar Script

FullCalendar is a jQuery plugin that provides a full-sized, drag & drop calendar like the one below.

It uses AJAX to fetch events on-the-fly for each month and is easily configured to use your own feed format (an extension is provided for Google Calendar). It is visually customizable and exposes hooks for user-triggered events (like clicking or dragging an event). It is open source and dual licensed under the MIT or GPL Version 2 licenses.

There are 4 files that you need to use in your HTMLs:

- fullcalendar.css
- fullcalendar.print.css
- jquery-ui-1.8.23.custom.min.js
- fullcalendar.min.js

In addition to these 4 files you need to initialize the script. Please, open calendar.html to see the initialization script.

If you need more information, visit the FullCalendar page.

Tabs

Here is a tab shortcode with two tabs. Add active property to tab-title and related tab-pane to make it active on page load.

[tabset]
	[tab_head]
		[tab_title active="active" sequence="1"]First Tab Title[/tab_title]
		[tab_title sequence="2"]Second Tab Title[/tab_title]
	[/tab_head]
	[tab_content]
		[tab_pane active="active" sequence="1"]Content for tab1.[/tab_pane]
		[tab_pane sequence="2"]Content for tab2.[/tab_pane]
	[/tab_content]
[/tabset]

Buttons

[button button_text="My button" button_link="" button_type="" button_size="btn-large" bold_text="yes"]

Button Colors : (7 colors avalaible), use "button_type" property for button color. Available colors:

  • White: Leave blank, default color is white.
  • Blue: button_type="btn-primary"
  • Sky Blue: button_type="btn-info"
  • Green: button_type="btn-success"
  • Yellow: button_type="btn-warning"
  • Red: button_type="btn-danger"
  • Black: button_type="btn-inverse"

Button Sizes; use "button_size" property, avalaible size;

  • Normal sized button: Leave blank, default size is normal.
  • Big sized button: button_size="btn-large"

Bold text in button: you can make button text bold, use bold_text property;

  • Normal text button: Leave blank, default text is normal.
  • Bold text button: bold_text="yes"

Alerts

You can create 4 types of alerts;

[alert alert_type="" closable_alert="x" alert_title="Warning!"]Best check yo self, you're not looking too good.[/alert]
[alert alert_type="alert-error" closable_alert="x" alert_title="Oh snap!"]Change a few things up and try submitting again.[/alert]
[alert alert_type="alert-success" closable_alert="x" alert_title="Well done!"You successfully read this important alert message.[/alert]
[alert alert_type="alert-info" closable_alert="x" alert_title="Heads up!"]This alert needs your attention, but it's not super important.[/alert]

Properties for alert shortcode;

  • alert_type : Decides alert's type (error,success,info or leave blank for default alert type)
  • closable_alert : is the text for closing the alert, you can enter x or leave blank for not closable alerts.
  • alert_title : is the bold text at the beginnnig of the alert's text.
